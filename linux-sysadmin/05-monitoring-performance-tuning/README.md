# Linux Monitoring & Performance Tuning: CPU, Memory, Disk, Limits

## What you will build
A monitoring + tuning workflow:
- establish baseline metrics
- identify top resource consumers
- disk usage analysis
- adjust a safe system limit and validate

## Why this matters
Monitoring and performance are daily production tasks.

## Skills
- `htop`, `free`, `df`, `du`
- sysstat: `iostat`, `vmstat`
- basic limits: `ulimit`, `limits.d`

## Success criteria
- Baseline captured (screenshots)
- Disk hog identified
- Limit changed and verified in a new session
